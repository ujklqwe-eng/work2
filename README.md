# Telegram-бот «Учёт работы техники и персонала»

Бот с мини-приложением (Web App) для сбора отчётов по объектам, видам работ,
технике и персоналу. Каждый отчёт сохраняется строкой в Excel-таблицу.

## Структура папки

```
telegram-bot/
├── bot.py              — код бота (запускается на вашем компьютере/сервере)
├── requirements.txt    — зависимости Python
├── webapp/             — форма отчёта (заливается на GitHub Pages)
│   ├── index.html
│   ├── style.css
│   └── script.js
└── README.md           — этот файл
```

## Шаг 1. Разместить форму (webapp) на GitHub Pages

1. Создайте публичный репозиторий на github.com (или используйте существующий)
2. Загрузите туда все 3 файла из папки `webapp/` (index.html, style.css, script.js) —
   именно содержимое папки, а не саму папку
3. В репозитории: **Settings → Pages → Source: Deploy from a branch → main → / (root) → Save**
4. Через 1–2 минуты сайт будет доступен по адресу вида:
   `https://ваш-логин.github.io/название-репозитория/`
5. Откройте эту ссылку в браузере — должна показаться форма отчёта

## Шаг 2. Создать бота в Telegram

1. Напишите **@BotFather** → `/newbot` → следуйте инструкциям
2. Скопируйте токен вида `123456789:AAH...`

## Шаг 3. Узнать свой Telegram ID (для прав администратора)

1. Напишите **@userinfobot** → `/start`
2. Скопируйте число — это ваш ID

## Шаг 4. Настроить bot.py

Откройте `bot.py` и замените:

```python
BOT_TOKEN = "ВАШ_ТОКЕН_БОТА"                          # ваш токен из шага 2
WEBAPP_URL = "https://ujklqwe-eng.github.io/work2/"   # ваша ссылка из шага 1
ADMIN_IDS = [123456789]                                # ваш ID из шага 3
```

## Шаг 5. Установить зависимости и запустить

В терминале, находясь в папке `telegram-bot`:

```bash
pip install -r requirements.txt
python bot.py
```

Если в терминале появилось `Start polling` — бот запущен и работает.
Терминал должен оставаться открытым, пока бот работает.

## Как пользоваться ботом

- **/start** — показывает кнопку «📋 Заполнить отчёт», открывающую форму
- Мастер заполняет ФИО, дату, один или несколько объектов, по каждому —
  виды работ, технику (с госномерами) и персонал
- После отправки отчёт сохраняется строкой в файл `reports.xlsx`
  (создаётся автоматически рядом с `bot.py`)
- **/excel** (только для админов из списка `ADMIN_IDS`) — присылает
  файл `reports.xlsx` со всеми отчётами прямо в чат

## Важно

- Файл `reports.xlsx` хранится локально, там же, где запущен `bot.py`.
  Если удалить/потерять этот файл — все отчёты пропадут, делайте бэкапы.
- Пока бот выключен (закрыт терминал/остановлен процесс) — он не отвечает.
  Для круглосуточной работы бота его нужно разместить на сервере
  (VPS, Railway, Render и т.п.), а не держать запущенным на личном компьютере.
- Токен бота — как пароль. Никому его не показывайте и не публикуйте.
